# 休憩促進アプリ（rest-promoter-app）

過集中しやすい人が、自己判断を損なわずに適切な休憩を取れるよう促すローカル Web アプリケーション。

---

## 目的

集中すると休憩をないがしろにして疲労困憊しやすい人向けに、**強制ではなく内省を促す**仕組みを提供する。

- タイマー発火時にダイアログで状態の言語化を求める → 無視しにくい
- 最終的な判断（休憩 or 継続）は常にユーザーが行う
- 状態スコア・作業時間・負荷の記録を積み重ねて自己認識を高める

---

## できること

| 機能 | 内容 |
|------|------|
| カウントダウンタイマー | 15 / 30 / 45 / 60 / 75 / 90 分のプリセットから選択。SVG 円形プログレスで残り時間を表示 |
| タイマー発火ダイアログ | タイマー終了時に自動表示。状態スコア確認 → 休憩 or スヌーズを選択 |
| 自己申告 | タイマーに関係なく任意のタイミングで状態を記録 |
| 強制休憩モード | 累積作業時間が 180 分を超えた状態でタイマー発火時は休憩 15 分のみ選択可 |
| 音声通知 | タイマー発火・休憩終了時に日本語で読み上げ（Web Speech API 対応ブラウザのみ） |
| 累積作業時間バー | 最後に休憩してからの連続作業時間を表示。135 分で黄色、180 分で赤 |
| 本日の統計 | 本日の総作業時間・総休憩時間をリアルタイム表示 |
| グラフ | 当日の状態スコア推移（折れ線）・連続作業時間（棒グラフ）を表示 |
| ログ記録 | 全記録を CSV ファイルに保存。pandas で分析可能 |

## 動作手順

### 対応環境

**Windows 10 / 11 のみ**（macOS・Linux は非対応）

推奨ブラウザ：Chrome または Edge（音声通知に Web Speech API が必要）

### 事前インストール

**uv**（Python パッケージマネージャー）のみ必要。Python は uv が自動管理するため別途インストール不要。

PowerShell で以下を実行：

```powershell
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"
```

インストール確認：

```powershell
uv --version
```

### 起動

1. リポジトリのフォルダを開く
2. `start.bat` をダブルクリック

自動的に：
- ポート 8000 の競合プロセスを終了
- Python 仮想環境（`.venv`）を再作成
- uvicorn サーバーを起動
- ブラウザで `http://127.0.0.1:8000` を開く

サーバーログは「REST Promoter」という別ウィンドウに表示される。アプリを終了する場合はそのウィンドウを閉じる。

### 画面操作の概要

```
[タイマー円形表示]
  作業中：青い円弧 → 残り時間がゼロになるとダイアログが自動表示
  休憩中：緑の円弧 → 休憩終了時にダイアログが自動表示

[累積作業時間バー]
  最後に休憩してからの連続作業時間。180 分で赤くなり警告バナーが表示

[本日の統計]
  本日の総作業時間・総休憩時間

[ボタン]
  開始 ……… タイマー開始。初回は作業内容入力ダイアログが表示される
  停止 ……… タイマー一時停止
  リセット …… タイマーを初期状態に戻す（累積時間はリセットされない）
  自己申告 … 任意のタイミングで状態を記録

[タイマー間隔]
  15 / 30 / 45 / 60 / 75 / 90 分から選択。現在の設定がハイライト表示。
  休憩中は変更不可

[グラフ・ログ]
  当日の状態スコア推移・連続作業時間グラフ
  直近 8 件のログ一覧
```

#### ダイアログの流れ

| 状況 | ステップ |
|------|---------|
| 初回起動 / リセット後の初回 | 作業内容 → 負荷 → 状態スコア |
| タイマー発火 / 自己申告（通常時） | 状態スコア → 休憩 or 継続の選択 → （休憩）休憩時間選択 / （継続）作業内容・負荷・スヌーズ時間 |
| タイマー発火（累積 180 分超） | 状態スコア → 休憩 15 分固定 |

## 開発方法

### Claude Code による AI 支援開発

このプロジェクトは [Claude Code](https://claude.ai/code) を使って開発している。

Claude Code を使うことで、コードの生成・修正・レビューをチャット形式で行える。

```bash
# Claude Code のインストール（Node.js が必要）
npm install -g @anthropic/claude-code

# プロジェクトフォルダで起動
claude
```

### ステアリングドキュメント（`.steering/`）

機能追加・修正の際は、`.steering/` 以下に作業単位のドキュメントを作成する。

```
.steering/
└── YYYYMMDD-開発タイトル/
    ├── requirements.md   # 要求内容
    ├── design.md         # 設計
    ├── tasklist.md       # タスクリスト
    └── decisions.md      # 意思決定ログ
```

- 各ファイルを作成するたびにレビューを行い、承認後に次のファイルへ進む
- 実装完了後も履歴として保持する
- アプリ全体の設計は `docs/` 以下の永続ドキュメントで管理する

### GitHub

```bash
# 変更を確認してコミット
git status
git add <ファイル>
git commit -m "変更内容の説明"
```

## 注意事項

- **ポート 8000 を使用する。** 他のアプリが使用中の場合は `start.bat` が自動的に終了させる
- **`.venv` は起動のたびに再作成される。** devcontainer（WSL2）との競合を防ぐための仕様。uv のキャッシュにより再作成は数秒で完了する
- **データは `data/log.csv` に保存される。** 削除すると記録が失われる
- **同時に複数のブラウザタブで開かない。** タイマー状態はサーバーで一元管理しているため、複数タブで操作すると意図しない動作になる場合がある
- **音声通知はブラウザの操作前には鳴らない。** ブラウザのオートプレイポリシーにより、ページを開いた直後（ユーザー操作なし）は音声が再生されない場合がある
