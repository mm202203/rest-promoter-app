# 要求定義：UX改善（ダイアログ・音声・タイマー）

## 概要

ユーザーの動作確認フィードバックをもとに、以下の5点を改善する。

---

## 1. ダイアログフロー改善（休憩時の作業入力省略）

### 変更内容

`timer` / `self` / `force` モードのダイアログフローを再設計する。
休憩を選択した場合、作業内容・作業負荷の入力を省略する。

### 新フロー

#### timer / self モード

```
Step 1: 状態スコアを選択（1〜5）
Step 2: 次のアクションを選択（「休憩する」or「作業を続ける」）
  → 休憩を選択:
      Step 3: 休憩時間を選択（5 / 10 / 15 / 60 分）→ 完了（計3ステップ）
  → 作業を続けるを選択:
      Step 3: 作業内容を確認・修正
      Step 4: 作業負荷を選択（1〜5）
      Step 5: スヌーズ時間を選択 → 完了（計5ステップ）
```

#### force モード

```
Step 1: 状態スコアを選択（1〜5）
Step 2: 休憩時間を選択（15分のみ）→ 完了（計2ステップ）
```

- 継続の選択肢なし（休憩時間選択のみ表示）

#### first モード（変更なし）

```
Step 1: 今回の作業内容を入力
Step 2: 作業負荷を選択（1〜5）
Step 3: 状態スコアを選択（1〜5）→ 完了（計3ステップ）
```

### ステップインジケーター（ドット）の動的更新

- timer / self の場合、Step 2 で選択するまで総ステップ数が確定しない
- Step 2 でいずれかのアクションを選択した時点で、ステップインジケーターのドット数と「次へ/完了」ラベルを動的に更新する
  - 休憩ルート選択 → ドット3個、Step 3 が最終ステップ（「完了」ラベル）
  - 継続ルート選択 → ドット5個、Step 5 が最終ステップ（「完了」ラベル）
- Step 2 選択前は仮のドット（timer/self はとりあえず5個、または未確定表示）でも可

### アドバイス取得・表示

- timer / self モード: Step 1 完了後に `POST /advice` を非同期で取得し、**継続ルートの Step 3（作業内容確認）の先頭**でバナー表示する
- **休憩ルートではアドバイスバナーを表示しない**（取得はしても表示しない）
- force モード: アドバイスの**取得も表示も行わない**（2ステップで完結するため不要）

### その他制約

- 休憩ルートでは `load` は `lastState.prev_load`（前回値）を引き継ぎ、`POST /record` の payload に含めてログに記録する
- `dialogData.action` を Step 2 で設定し、それ以降のステップ描画・バリデーション・submit に使用する

## 2. 音声通知

### 変更内容

タイマー発火時・休憩終了時にブラウザの Web Speech API（`SpeechSynthesis`）で日本語音声を再生する。

### 発話タイミングとメッセージ

| タイミング | 条件 | メッセージ |
|-----------|------|-----------|
| タイマー発火時 | `dialog_triggered === true` かつ `lastState.is_breaking === false` | 「作業時間が終わりました。今の状態を教えてください。」 |
| 休憩終了時 | `dialog_triggered === true` かつ `lastState.is_breaking === true` | 「休憩が終わりました。今の状態を教えてください。」 |

- `lastState` は直前のポーリング結果（`dialog_triggered` が立つ前の状態）なので、`is_breaking` の値で発話内容を切り替えられる
- `self` モード（自己申告）: ユーザー自発のためダイアログは開くが音声は再生しない（`dialog_triggered` を経由しないため自然に対象外になる）
- `force` モード（累積180分超）: タイマー発火扱い（`is_breaking === false`）なので「作業時間が終わりました」が再生される

### フォールバック

- ブラウザのオートプレイポリシー（ユーザー操作前は音声が鳴らない場合がある）を考慮し、失敗しても無視する（`try/catch` で握り潰す）
- 音声は日本語（`lang: 'ja-JP'`）を指定する
- 動作しない環境ではサイレントにフォールバックする（タスクリストに「動作確認が取れない場合はサイレントフォールバックを確認する」と明記）

## 3. SVG 円形プログレスバー

### 変更内容

タイマー残り時間をデジタル表示（`MM:SS`）と SVG 円形プログレスバーで一体化して表示する。

### 実装仕様

- SVG の `<circle>` 要素の `stroke-dashoffset` を使って円弧の長さで進行率を表現する
- 円のサイズ: `width="200" height="200"`、半径 `r="88"`（中心 `cx="100" cy="100"`）
- 線の太さ（`stroke-width`）: `12`
- 背景円（トラック）: `stroke="#e0e0e0"`
- 進行円: 作業中 `stroke="#1976d2"`、休憩中 `stroke="#4caf50"`
- 進行率の計算:
  - 作業中: `(timer_duration - remaining) / timer_duration`
  - 休憩中: `(breakDuration - remaining) / breakDuration`
    - `breakDuration` はフロントエンドで「休憩開始時点の `remaining` 値」を変数に保存して使用する（`is_breaking` が `false → true` に変わった瞬間に記録）
- デジタル表示（`MM:SS`）は SVG の `<text>` 要素で円の中心に配置する
- セッション経過時間表示（`セッション経過: MM:SS`）は円の下に維持する

### DOM 変更

- 既存の `#timer-display`（テキスト要素）を SVG 要素に置き換える
- `#session-elapsed` はそのまま SVG の下に配置する

## 4. 作業時間プリセット・休憩時間追加

### 4-1. 作業時間プリセットボタン

現状のテキスト入力（1〜120分）をプリセットボタンに置き換える。

| プリセット | 分数 |
|-----------|------|
| 15分 | 15 |
| 30分 | 30 |
| 45分 | 45 |
| 60分 | 60 |
| 75分 | 75 |
| 90分 | 90 |

- 現在設定中の値（`state.timer_duration / 60`）に対応するボタンをハイライト（`.active` クラス付与）する
- テキスト入力フィールド（`#input-duration`）と「変更」ボタン（`#btn-config`）は廃止する
- プリセットボタンをクリックしたら即座に `POST /config` を送信する

### 4-2. 休憩時間 60 分追加

ダイアログの休憩時間選択肢に「60分」を追加する。

| モード | 選択肢 |
|--------|--------|
| timer / self（休憩ルート） | 5 / 10 / 15 / 60 分 |
| force | 15分のみ（変更なし） |

## 5. CSV へのセッション時刻追加

### 変更内容

CSV に以下の 2 カラムを追加する。

| カラム名 | 内容 |
|---------|------|
| `session_start` | 作業セッション開始時刻（ISO 8601 形式、JST） |
| `session_end` | 作業セッション終了時刻（ISO 8601 形式、JST） |

### アクション別の記録内容

| action | session_start | session_end |
|--------|--------------|-------------|
| `start` | ログ記録時刻と同値（セッション開始 = 今） | 空文字（まだ終了していない） |
| `skip` | `TimerState.session_start_time`（スナップショット取得時点の値） | ログ記録時刻と同値 |
| `rest` | `TimerState.session_start_time`（スナップショット取得時点の値） | ログ記録時刻と同値 |

- `append_log()` は `get_state_snapshot()` → `append_log()` → `do_record()` の順で呼ばれる
  - `skip` / `rest` ではスナップショット時点の `session_start_time` を記録し、その後 `do_record()` で `session_start_time` を更新する
  - `start` では `do_record()` 実行前なのでスナップショットの `session_start_time` は未設定（空）のため、`append_log()` 呼び出し元でログ記録時刻を `session_start` として渡す

### バックエンド変更

- `TimerState` に `session_start_time: str = ""` フィールドを追加
- `do_record()` で更新するタイミング:
  - `action="start"`: `session_start_time = now`（JST ISO 8601）
  - `action="skip"`: `session_start_time = now`（新セッション開始）
  - `action="rest"`: `session_start_time = ""`（セッション終了、次の start まで空）
- `log_service.py` の `COLUMNS` に `session_start` / `session_end` を追加
- `append_log()` のシグネチャに `session_start: str` / `session_end: str` を追加

### 既存 CSV の後方互換性

- 既存の `data/log.csv` には `session_start` / `session_end` カラムが存在しない
- `read_logs()` で pandas 読み込み後、欠損カラムを空文字で補完する（`df[col] = df.get(col, "")`）

---

## 受け入れ条件

1. timer/self ダイアログで「休憩する」を選択すると、作業内容・負荷入力がスキップされ、休憩時間選択のみで完了する
2. timer/self ダイアログで「作業を続ける」を選択すると、作業内容→負荷→スヌーズ時間のフローになる
3. force ダイアログは Step 1（状態）→ Step 2（15分休憩のみ）の 2 ステップになる
4. タイマー発火・休憩終了時に指定の音声が再生される（対応ブラウザ）
5. 動作しない環境でも無音でフォールバックし、エラーが出ない
6. タイマー表示が SVG 円形プログレスと一体化されている（作業中: 青、休憩中: 緑）
7. 作業時間設定が 15/30/45/60/75/90 分のプリセットボタンになっており、現在値がハイライトされる
8. ダイアログの休憩選択肢に「60分」が追加されている
9. CSV の各行に `session_start` / `session_end` が記録されている（skip / rest 行は両方、start 行は session_start のみ）
10. 既存の CSV データを読み込んでもエラーが出ない
