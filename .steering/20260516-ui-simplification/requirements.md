# requirements.md — UI シンプル化

## 概要

画面をシンプル化し、必要最小限の要素に絞る。

---

## 変更要件

### 1. ボタン構成の整理

- メインボタンを3つに絞る：**開始** / **立ち止まる** / **終了**
- 「自己申告」を「立ち止まる」にリネーム（機能変更なし）
- 「本日の作業終了」を「終了」にリネーム（機能変更なし）
- 「停止」ボタン：引き続き非表示（DOM・JS制御は現状維持）
- 「リセット」ボタン：引き続き非表示（同上）
- 「日報を出力」ボタン：最近の記録セクションのヘッダー右端に小型ボタンとして移動

### 2. 最近の記録の折りたたみ

- セクションをデフォルトで折りたたんだ状態にする
- ヘッダーをクリックすることで展開・折りたたみを切り替える
- ヘッダー右端に展開状態を示すシェブロンアイコン（▼/▲）を表示
- CSS `max-height` transition によるスムーズなアニメーション
- ページ再読み込みのたびに閉じた状態に戻る（localStorage不使用）

### 3. 「立ち止まる」ボタンのツールチップ

- ホバー時にツールチップを表示
- 表示内容：「今の状態を記録して次のアクションを決めます」
- CSS のみで実装（`:hover::after` 疑似要素）
- ボタン上部に吹き出し風で表示

### 4. グラフ点ホバーでの作業時間表示

- 状態スコアグラフの各点にホバーしたとき、作業時間を確認できる
- Chart.js の tooltip をカスタマイズして表示
- 表示内容：`状態: X / 作業時間: Y分`
- データ構造：各点を `{y: state, session_min: m}` 形式に変更し `parsing.yAxisKey` で y 軸を指定

### 5. 期間ボタンのデザイン刷新

- セグメントコントロール風デザインに変更
  - 薄グレー背景のコンテナに角丸ピルボタンを並べる
  - アクティブ時：白背景＋ドロップシャドウ
  - 非アクティブ時：透明背景
- 位置：chart-title-row の右端（現状の配置を維持）

---

## 制約

- バックエンドAPIとの接続（ポーリング・ダイアログ送信）は引き続き正常に動作すること
- Python バックエンドの変更は不要
- `btn-pause`・`btn-reset` のDOM要素とJSイベントリスナーは保持する（非表示のまま）

---

## ユーザーストーリー

- ユーザーとして、画面が整理されていてどのボタンを押せばいいか迷わずに使いたい
- ユーザーとして、「立ち止まる」ボタンの意味をホバーで確認して安心して押したい
- ユーザーとして、グラフの点にホバーして詳細な作業時間を知りたい
- ユーザーとして、最近の記録は普段は邪魔にならないよう非表示にしておきたい

---

## 受け入れ条件

- [ ] メインボタンが「開始・立ち止まる・終了」の3つに絞られている
- [ ] 「日報を出力」ボタンが最近の記録セクション近くにある
- [ ] 最近の記録がデフォルトで折りたたまれている
- [ ] ヘッダークリックで展開・折りたたみができる
- [ ] 「立ち止まる」ホバー時にツールチップが表示される
- [ ] グラフ点ホバーで「状態: X / 作業時間: Y分」が表示される
- [ ] 期間ボタンがセグメントコントロール風に見える
- [ ] ポーリング・ダイアログ送信が正常に動作する
