# 要求内容：初回実装

## 概要

`docs/` 配下の設計ドキュメントをもとに、休憩促進アプリ（MVP フェーズ1）を初回実装する。
ユーザーストーリー・ドメイン詳細は `docs/product-requirements.md` を参照。

---

## 実装する機能

### 1. バックエンド（FastAPI）

| 機能 | 詳細 |
|------|------|
| タイマー状態管理 | `TimerState`（`@dataclass`）のシングルトン管理、`threading.Lock` による排他制御 |
| バックグラウンドスレッド | アプリ起動時（lifespan）に常駐。1秒ごとに状態更新・タイマー発火判定を行う |
| タイマー操作 API | `GET /state` `POST /start` `POST /pause` `POST /reset` `POST /config` |
| ダイアログ API | `POST /record` `POST /dialog/ack` |
| ログ API | `GET /logs`（timestamp 昇順） |
| CSV 永続化 | `data/log.csv` への追記・読み込み。初回起動時に自動生成 |
| ダイアログモード判定 | `first` / `timer` / `force` の判定ロジック（`session_service`） |
| アドバイス分岐 | 状態スコア・負荷・`session_elapsed` の条件評価（4段階優先度、`session_service`） |
| CSV インジェクション対策 | `task` 先頭文字が `=+-@` の場合にシングルクォートを付加 |
| 静的ファイル配信 | FastAPI `StaticFiles` で `static/` をマウント（CORS 設定不要） |

### 2. フロントエンド（HTML/CSS/JS）

| 機能 | 詳細 |
|------|------|
| タイマー表示 | 残り時間（MM:SS）の表示。通常時・休憩中で表示を切り替える |
| セッション経過時間表示 | 現セッションの経過時間を表示。休憩中はカウント停止 |
| 累積バー | `accum_elapsed` の進捗表示。135分で黄色・180分で赤 |
| 休憩中画面状態 | 「休憩中 MM:SS」表示・操作ボタン無効化・累積バーリセット後表示 |
| タイマー操作 | 開始・停止・リセットボタン。休憩中は無効化 |
| タイマー間隔変更 | 入力フィールドと変更ボタン |
| 自己申告ボタン | 常駐ボタン。クライアント側で直接 `self` ダイアログを起動。ダイアログ表示中は無効化 |
| ダイアログ（オーバーレイ） | `first` / `timer` / `self` / `force` の4モードに対応したステップUI |
| ステップ間ナビゲーション | ダイアログ内の「次へ」「戻る」ボタン。ステップインジケーター表示 |
| ダイアログ内グラフ | 状態スコア推移グラフ（今回入力値をプレビュー表示）をダイアログ Step1 に表示 |
| 状態スコア推移グラフ | Chart.js 折れ線グラフ。直近30件・点の色分け（1〜2=赤、3=グレー、4〜5=緑） |
| 連続作業時間グラフ | Chart.js 棒グラフ。直近30件・`first` レコード（`session_min=0`）除外 |
| ログ一覧 | 直近8件を timestamp 降順で表示 |
| 接続エラー表示 | `fetch` 失敗時に画面上部にバナーを表示。復帰時は自動で非表示 |
| ポーリング | `setInterval` 1秒。二重実行防止フラグ・ダイアログ重複発火防止フラグを持つ |

### 3. 起動・環境

| 機能 | 詳細 |
|------|------|
| `pyproject.toml` | 依存関係定義（fastapi / uvicorn / pandas）+ dev 依存（ruff） |
| `start.bat` | uvicorn 起動（127.0.0.1:8000）+ ブラウザ自動オープン |
| `git init` | ローカルリポジトリ初期化・`.gitignore` 設定 |

## 受け入れ条件

### `docs/product-requirements.md` からの主要条件（転記）

- [ ] タイマー発火時に画面を覆うオーバーレイが表示され、別の操作をしない限り閉じない
- [ ] 状態スコアが 1〜2 の場合、ダイアログ枠に警告色（赤ボーダー）が表示される
- [ ] 累積作業時間が 135分 に達したとき累積バーが黄色になる
- [ ] 累積作業時間が 180分 に達したとき累積バーが赤になる
- [ ] `force` モードはタイマー発火時のみ発動する（タイマー計測中のリアルタイム割り込みはしない）
- [ ] `force` モード時はスヌーズ選択肢が非表示になり、休憩時間は 15分 固定となる
- [ ] `accum_elapsed` が上限超過中に自己申告ボタンを押した場合は `self` モードで開く
- [ ] 休憩を実際に選択したときのみ、`accum_elapsed` がゼロにリセットされる
- [ ] `first` モードはアプリ起動後の最初の開始、およびリセット後の再開始時に発動する
- [ ] `first` モードで記録した内容はグラフとログに反映される
- [ ] 前回の作業内容・負荷がデフォルト値として表示される
- [ ] 作業内容テキストが未入力の状態で次へ進もうとした場合、エラー表示してステップを進めない
- [ ] ログが CSV ファイルに書き込まれ、アプリ再起動後も参照できる
- [ ] 休憩タイマーが 0 になったら自動で `timer` フローのダイアログを起動する
- [ ] タイマー間隔を変更でき、変更後のセッションから新しい時間が適用される

### 初回実装固有の条件

- [ ] `start.bat` をダブルクリックするだけでブラウザが開きアプリが使える状態になる
- [ ] `data/log.csv` が存在しない状態から初回起動して正常に動作する
- [ ] タイマー間隔を 1〜120分の範囲で変更でき、範囲外は 422 エラーで弾かれる
- [ ] サーバーを停止してもログデータが `data/log.csv` に残っている
- [ ] `uv run ruff check .` がエラーなしで通る
- [ ] `uv run ruff format --check .` を実行して差分が出ない（フォーマット済みであること）
- [ ] `.gitignore` に `data/log.csv`・`__pycache__/`・`.venv/` が含まれている

---

## 制約事項

- Python 3.11 以上、uv 使用
- 外部サービス・クラウド不使用
- フロントエンドはビルドステップなし（Vanilla JS + Chart.js CDN、バージョン 4.x 固定）
- `StaticFiles` で `static/` を配信する。同一オリジンのため CORS 設定不要
- 複数タブの同時アクセスは非サポート
- サーバー再起動時のタイマー状態復元は MVP 対象外
- 自動テストは MVP 対象外（`docs/development-guidelines.md` の手動チェックリストで確認）
