# ユビキタス言語定義

ドメイン内で使われる用語を統一し、ドキュメント・コード・会話のすべてで同じ言葉を使う。

---

## ドメイン用語

### 作業・休憩に関する用語

| 日本語 | 英語（コード上） | 定義 |
|--------|----------------|------|
| 作業セッション | session | タイマーを開始してから発火するまでの1サイクル。スヌーズ後も新たなセッションが始まる |
| セッション経過時間 | `session_elapsed` | 現在のセッションで作業した時間（秒）。スヌーズ・休憩のどちらでもリセットされる |
| 累積作業時間 | `accum_elapsed` | 最後に休憩を取ってから連続して作業した合計時間（秒）。**休憩を取ったときのみリセット**。スヌーズではリセットされない |
| 休憩 | rest | ユーザーが作業を中断し、設定した時間（5/10/15分）だけ意識的に休む行動。`accum_elapsed` をリセットする |
| スヌーズ | snooze / skip | 休憩を先送りにし、設定した時間（15/30/45/60分）後に再度介入を受ける選択。`accum_elapsed` はリセットしない |
| 過集中 | hyperfocus | 特定の作業に没入しすぎて時間感覚や疲労感を失う状態。本アプリが主に対処しようとする課題 |

### タイマーに関する用語

| 日本語 | 英語（コード上） | 定義 |
|--------|----------------|------|
| タイマー間隔 | `timer_duration` | カウントダウンの設定時間（秒）。デフォルト 3600秒（60分）。ユーザーが変更可能（1〜120分） |
| カウントダウン残り時間 | `remaining` | 現在のタイマーが発火するまでの残り秒数 |
| タイマー発火 | timer fire | `remaining` が 0 になり、介入ダイアログが表示されるイベント |
| 休憩タイマー | break timer | 休憩を選択した後に動くカウントダウン。`is_breaking = true` 中に動作する。終了後に自動で `timer` ダイアログを発火する |
| ポーリング | polling | UI が 1秒ごとに `GET /state` を呼び出してサーバーの状態を同期する仕組み |

### 状態・スコアに関する用語

| 日本語 | 英語（コード上） | 定義 |
|--------|----------------|------|
| 状態スコア | `state` | ユーザーが自己申告するコンディションの5段階評価（1=かなり悪い 〜 5=かなりよい） |
| 作業負荷 | `load` | ユーザーが行っている作業の認知的重さの5段階評価（1=軽い 〜 5=重い） |
| 作業内容 | `task` | ユーザーが今行っている作業の説明テキスト |
| 警告状態 | warn | `accum_elapsed` が 135分〜179分の状態。累積バーが黄色になる |
| 上限状態 | danger | `accum_elapsed` が 180分以上の状態。累積バーが赤になり、`force` モードが有効になる |

## ダイアログに関する用語

### ダイアログモード

| モード名 | コード上の値 | 発動条件 | 特徴 |
|---------|------------|---------|------|
| 初回モード | `"first"` | アプリ起動後の最初の開始、またはリセット後の再開始 | 作業内容・負荷・状態スコアを入力。休憩/スヌーズ選択なし。完了後にタイマーが開始する |
| タイマーモード | `"timer"` | タイマー発火時、かつ `accum_elapsed` が 180分未満 | 状態スコア確認 → 作業内容 → 負荷 → 休憩/スヌーズ選択の4ステップ |
| 自己申告モード | `"self"` | 自己申告ボタン押下時（`accum_elapsed` の状態を問わず） | `timer` と同フロー。冒頭文言のみ「自己申告：」に変更。クライアント側で直接ダイアログを開く |
| 強制休憩モード | `"force"` | タイマー発火時、かつ `accum_elapsed` が 180分以上 | 警告バナー表示。スヌーズ選択肢を非表示。休憩15分固定 |

### ダイアログ操作に関する用語

| 日本語 | 英語（コード上） | 定義 |
|--------|----------------|------|
| 介入 | intervention | タイマー発火またはユーザー操作により、作業を一時中断させてダイアログを表示すること |
| ダイアログ発火フラグ | `dialog_triggered` | サーバーがダイアログ表示をUIに通知するためのフラグ。UIが検知して `POST /dialog/ack` を送るまで `true` のまま |
| ACK（確認応答） | `dialog/ack` | UIがダイアログを受け取ったことをサーバーに通知する操作。二重発火を防ぐ |
| ステップ | step | ダイアログ内の各入力画面。`first` は3ステップ、`timer`/`self`/`force` は4ステップ |

## アドバイスに関する用語

| 日本語 | 定義 |
|--------|------|
| アドバイス | ダイアログの状態スコア入力後に表示されるメッセージ。ユーザーの状態・負荷・経過時間から自動判定される |
| 警告バナー | アドバイスの中でも強い注意を促す赤または黄色の帯表示。ユーザーが見落としにくい形で強調する |
| アドバイス分岐 | 状態スコア・作業負荷・`session_elapsed` の条件を評価し、4段階の優先度で最適なアドバイスを選択するロジック |

## UI/UX 用語

| 日本語 | 英語（コード上） | 定義 |
|--------|----------------|------|
| 累積バー | accum bar | `accum_elapsed` の進行を視覚化するプログレスバー。180分を上限として表示し、状態に応じて色が変わる |
| タイマー表示 | timer display | カウントダウン残り時間を `MM:SS` 形式で表示するUI要素 |
| セッション経過表示 | session elapsed display | 現セッションの経過時間を表示するUI要素 |
| 自己申告ボタン | self-report button | ユーザーが任意のタイミングで `self` モードダイアログを起動するための常駐ボタン |
| オーバーレイ | overlay | 画面全体を覆い、下の要素への操作を遮断する半透明の背景。ダイアログ表示時に使用 |
| 状態スコア推移グラフ | state score chart | 記録ごとの状態スコアを折れ線グラフで表示。直近30件。点の色で値を視覚化する |
| 連続作業時間グラフ | session duration chart | 各セッションの作業時間（分）を棒グラフで表示。直近30件。`first` モードレコードは除外 |
| ステップインジケーター | step indicator | ダイアログ内で現在のステップ番号を示すドット列（例: `●──○──○──○`） |

## ログ・データに関する用語

| 日本語 | 英語（コード上） | 定義 |
|--------|----------------|------|
| ログレコード | log record | ダイアログ完了時に CSV に書き込まれる1行分のデータ |
| アクション | `action` | ダイアログでユーザーが選んだ行動。`"start"`（firstモード完了）/ `"rest"`（休憩）/ `"skip"`（スヌーズ）の3値 |
| 休憩時間 | `break_min` | 休憩を選んだ場合の休憩時間（分）。`action = "rest"` 時のみ値を持つ |
| スヌーズ時間 | `snooze_min` | スヌーズを選んだ場合の次のタイマー時間（分）。`action = "skip"` 時のみ値を持つ |

## 使ってはいけない言葉・揺れの統一

以下の表現は混乱を招くため、統一する。

| 使わない表現 | 統一する表現 | 理由 |
|------------|------------|------|
| 「通知」「リマインダー」 | 「介入」「ダイアログ」 | 通知はOSの通知機能を想起させる。本アプリの仕組みはオーバーレイによる介入であり異なる |
| 「タイムアウト」 | 「タイマー発火」 | タイムアウトはエラーのニュアンスがある |
| 「強制」（force モード以外で使う） | 「介入」 | force モード以外では強制ではなく自律的な選択を促す設計のため |
| 「カウンター」 | 「タイマー」または「累積作業時間」 | 指す対象を明確にする |
| 「ポップアップ」 | 「ダイアログ」 | ブラウザのポップアップブロックと混同を避ける |
