# プロダクト要求定義書

## プロダクトビジョンと目的

過集中しやすいユーザーが、自己判断を損なわずに適切な休憩を取れるよう促すローカル Web アプリケーション。
「強制」ではなく **「自己認識の鏡」** を提供することで、持続可能な作業リズムを支援する。

「適切な休憩」とはユーザー自身が状況・状態に応じて判断するものであり、アプリはその判断を促す場を提供する。
タイマー間隔はデフォルト 60分 だが、ユーザーが変更可能とする。

### 背景・課題

- 過集中の性質があり、集中すると休憩をないがしろにして疲労困憊しやすい
- 通知・タイマーは「無視して続ける」が容易で効果が薄い
- 自分の疲労や状態の悪化に気づきにくい

### 解決アプローチ

単なるリマインダーではなく、**介入時に会話UIを出してユーザーに内省を促す**。
返答する過程で自分の状態を言語化させることで、無視しにくくする。
最終的な判断は常にユーザーに委ねる（強制ではなく自律的な選択）。

## ターゲットユーザーと課題・ニーズ

### ターゲットユーザー（ペルソナ）

**本アプリは作者本人（以下「ユーザー」）のための個人利用ツールである。**

| 属性 | 内容 |
|------|------|
| 職種・作業タイプ | ソフトウェア開発・論文読解・資料作成などの知的作業が中心 |
| 過集中の傾向 | 一度集中すると時間感覚を失いやすく、気づけば数時間経過していることがある |
| 休憩への抵抗感 | 「もう少しで区切りがつく」と感じやすく、休憩を先送りにしがち |
| 強制への反発 | 外部から強制されることを嫌う。自分で判断して行動したい |
| 技術リテラシー | Python / uv の操作が可能。ローカルサーバーの起動・管理ができる |
| 使用環境 | Windows PC、Chrome または Edge ブラウザ、終日デスクワーク中に常駐利用 |

### 課題・ニーズ

| 課題 | ニーズ |
|------|--------|
| 休憩タイミングを見失いやすい | 適切なタイミングで介入してほしい |
| 通知は無視しやすい | 無視しにくい仕組みが欲しい |
| 自分の状態悪化に気づきにくい | 自分の状態を客観視する手助けが欲しい |
| 強制されると反発する | 自律的に判断したい |

### MVPフェーズ（フェーズ1）

| 機能 | 概要 |
|------|------|
| タイマー | カウントダウンタイマー（デフォルト60分、変更可能）。開始・一時停止・リセット操作。タイマー発火時に介入ダイアログを自動表示 |
| 累積作業時間管理 | セッション経過時間と累積作業時間を分離管理。閾値（135分・180分）で UI 警告色変化 |

| 状態記録 | 作業内容・作業負荷（1〜5）・状態スコア（1〜5）・判断（休憩/スヌーズ）を収集 |
| アドバイス分岐 | 状態スコア・作業負荷・セッション経過時間に応じた警告バナーとアドバイス表示 |

| ログ表示 | 直近8件を画面に表示。CSV に永続化 |
| 自己申告ボタン | 画面常駐の小さいボタン。ユーザーが任意のタイミングでダイアログを起動できる |

## 成功の定義

### 定性的な成功状態

- タイマー発火時に必ずダイアログが表示され、ユーザーが何らかの応答をするまで作業を継続できない
- ユーザーが自分の状態（疲労・負荷）を言語化するプロセスを経ることで、内省が促される
- 累積作業時間のログが蓄積され、自分の作業パターンを振り返ることができる
- アプリを再起動してもデータが失われない

### 測定可能な成功指標

| 指標 | 目標 | 計測方法 |
|------|------|---------|
| 休憩選択率（rest率） | rest / (rest + skip) > 50% | CSV の `action` カラムで集計 |
| force モード発生回数 | 週あたり 3回以下 | CSV の `dialog_mode = 'force'` で集計 |
| アプリ継続利用日数 | ログが途切れず蓄積されている | CSV の `timestamp` の連続性で確認 |

## ビジネス要件

- 個人利用・自分用（配布不要）
- ローカル環境で完結（外部サービス不要）
- Windows 環境で `.bat` ファイルをダブルクリックするだけで起動
- 追加インストール不要（uv でパッケージ管理）

## ユーザーストーリー

1. **タイマー介入**
   > 作業者として、設定した時間が経過したときに自動でダイアログが表示され、今の状態を振り返りたい。
   > そうすることで、無意識に過集中が続くことを防げる。

2. **状態の言語化**
   > 作業者として、作業内容・負荷・状態スコアを入力するプロセスを通じて、自分の疲労度を客観視したい。
   > そうすることで、「まだ大丈夫」という思い込みに気づける。

3. **自己申告**
   > 作業者として、タスクの区切りなど自分のタイミングでもダイアログを起動したい。
   > そうすることで、タイマーを待たずに自律的に休憩を選択できる。

4. **累積作業時間の把握**
   > 作業者として、最後の休憩から何分連続で作業しているか常に確認したい。
   > そうすることで、知らないうちに長時間連続作業していることに気づける。

5. **強制休憩（force モード）**
   > 作業者として、累積作業時間が180分を超えた場合は休憩以外の選択肢を取り除いてほしい。
   > そうすることで、健康上の上限を自動的に守れる。

6. **ログの振り返り**
   > 作業者として、過去の状態スコアと作業時間のグラフを確認したい。
   > そうすることで、自分の作業パターンと状態変化の傾向を把握できる。

7. **タイマー時間の調整**
   > 作業者として、その日の作業内容や集中度に合わせてタイマー間隔を変更したい。
   > そうすることで、状況に応じた柔軟な休憩管理ができる。

## 受け入れ条件

- [ ] タイマー発火時に画面を覆うオーバーレイが表示され、別の操作をしない限り閉じない
- [ ] 状態スコアが 1〜2 の場合、ダイアログ枠に警告色（オレンジ〜赤）が表示される
- [ ] 累積作業時間が 135分 に達したとき累積バーが黄色になる
- [ ] 累積作業時間が 180分 に達したとき累積バーが赤になる
- [ ] `force` モードはタイマー発火時のみ発動する（タイマー計測中のリアルタイム割り込みはしない）
- [ ] `force` モード時はスヌーズ選択肢が非表示になり、休憩時間は 15分 固定となる
- [ ] `accum_elapsed` が上限超過中に自己申告ボタンを押した場合は `self` モードで開く
- [ ] 休憩を実際に選択したときのみ、`accum_elapsed` がゼロにリセットされる
- [ ] `first` モードはアプリ起動後の最初の開始、およびリセット後の再開始時に発動する
- [ ] `first` モードで記録した内容はグラフとログに反映される
- [ ] 前回の作業内容・負荷がデフォルト値として表示される
- [ ] 作業内容テキストが未入力の状態で次へ進もうとした場合、エラー表示してステップを進めない
- [ ] ログが CSV ファイルに書き込まれ、アプリ再起動後も参照できる
- [ ] 休憩タイマーが 0 になったら自動で `timer` フローのダイアログを起動する
- [ ] タイマー間隔を変更でき、変更後のセッションから新しい時間が適用される

## 機能要件

### タイマー

- カウントダウン時間はユーザーが変更可能。デフォルト 60分
- 変更可能な範囲：1〜120分（整数）
- タイマー間隔の変更は次のセッション開始時から反映（進行中のタイマーには影響しない）
- 開始・一時停止・リセット操作が可能
- タイマーは Python サーバー側のバックグラウンドスレッドで管理（UI はレンダリングのみ）
- UI は 1秒ごとに `GET /state` をポーリングして表示を同期
- サーバー再起動時はタイマー状態が初期化される（状態はメモリのみで管理。MVPでは再起動時の復元は対象外）
- 複数タブからの同時アクセスは想定外。単一タブでの使用を前提とする

### 累積作業時間管理

| 変数 | 意味 | リセットタイミング |
|------|------|--------------------|
| `session_elapsed` | 現在のタイマーセッションで作業した時間 | スヌーズ時・休憩時ともにリセット |
| `accum_elapsed` | 最後の休憩から一切休憩なしで作業した合計時間 | 休憩を取ったときのみリセット |

| 累積時間 | 状態 | UI 表示 |
|----------|------|---------|
| ～134分 | 通常 | 累積バー：通常色 |
| 135～179分 | 警告 | 累積バー：黄色 |
| 180分以上 | 上限 | 累積バー：赤色 |

### ダイアログモード

| モード | 発動条件 | 特徴 |
|--------|----------|------|
| `first` | アプリ起動後の最初の開始・リセット後の再開始 | 作業内容・負荷・状態スコアを入力。休憩/スヌーズ選択なし |
| `timer` | タイマー発火時（`accum_elapsed` が上限未満） | 状態確認 → 作業内容 → 休憩 or スヌーズ選択 |
| `self` | 自己申告ボタン押下時（`accum_elapsed` の状態を問わず） | `timer` と同フロー。冒頭文言のみ「自己申告：」に変更 |
| `force` | タイマー発火 + `accum_elapsed` が上限以上 | 警告バナー・スヌーズ非表示・休憩15分固定 |

#### ダイアログのステップフロー

**`first` モード（3ステップ）**

| ステップ | 内容 | 入力形式 |
|---------|------|---------|
| 1 | 「これからどんな作業をしますか？」作業内容の入力 | テキスト入力（必須・未入力で進行不可） |
| 2 | 「作業の負荷はどのくらいですか？」作業負荷の選択 | 1〜5 の5段階選択 |
| 3 | 「今の状態はどうですか？」状態スコアの選択 | 1〜5 の5段階選択 |
| 完了 | ログ記録してダイアログを閉じる。タイマー開始 | — |

**`timer` / `self` モード（4ステップ）**

| ステップ | 内容 | 入力形式 |
|---------|------|---------|
| 1 | 「今の状態はどうですか？」状態スコアの選択 + アドバイス表示 | 1〜5 の5段階選択 |
| 2 | 「作業内容を確認してください」作業内容の入力（前回値がデフォルト） | テキスト入力（必須・未入力で進行不可） |
| 3 | 「作業の負荷はどのくらいですか？」作業負荷の選択（前回値がデフォルト） | 1〜5 の5段階選択 |
| 4 | 「どうしますか？」休憩またはスヌーズの選択 | 休憩（5/10/15分）or スヌーズ（15/30/45/60分） |
| 完了 | ログ記録・タイマー再起動 | — |

**`force` モード（4ステップ、スヌーズなし）**

`timer` モードと同じステップだが、ステップ4の選択肢は休憩（15分固定）のみ。スヌーズ選択肢は表示しない。

### アドバイス分岐

複数条件が同時に成立する場合は **上から順に最初にマッチした条件を適用する**（優先度順）。

| 優先度 | 条件 | アドバイス・UI変化 |
|--------|------|-----------------|
| 1（最高） | 状態スコア 1〜2 | ダイアログ枠が赤ボーダー。強い警告バナー表示 |
| 2 | 状態スコア 3 かつ `session_elapsed` が 90分超 | 警告バナー表示。休憩を勧める |
| 3 | 負荷 4〜5 かつ `session_elapsed` が 45分超 | 警告バナー表示。認知負荷の蓄積を説明 |
| 4（最低） | その他 | 状態良好として継続を支持 |

### グラフ表示

- **状態スコア推移**：折れ線グラフ。直近 30件 の記録を表示。点の色：1〜2=赤、3=グレー、4〜5=緑
- **連続作業時間**：棒グラフ。直近 30件 のセッション作業時間（分）を表示
- ダイアログ内にも状態スコア推移グラフを表示（「今回」を追加した状態でプレビュー）

### ログ表示

- メイン画面に直近8件を表示（timestamp 降順）
- 各レコード：時刻・作業内容・負荷（ドット表示）・状態スコア（色付きラベル）・行動（休憩/継続）

### 自己申告ボタン

- メイン画面に常駐する小さいボタン
- ダイアログ表示中は無効化する（ダイアログが閉じてから使用可能）
- `accum_elapsed` が上限超過中でも `self` モードで開く（force モードにはならない）

## 非機能要件

| カテゴリ | 要件 |
|---------|------|
| **プラットフォーム** | Windows 10 以降 |
| **ブラウザ** | Google Chrome または Microsoft Edge（最新版） |
| **Python バージョン** | Python 3.11 以降 |
| **パッケージ管理** | uv |
| **起動方法** | `.bat` ファイルをダブルクリックで起動（uvicorn 起動 + ブラウザ自動オープン） |
| **ポート番号** | デフォルト 8000番（`http://localhost:8000`） |
| **ネットワーク** | localhost（127.0.0.1）のみバインド。外部からのアクセス不可 |
| **データ永続化** | CSV ファイルによるログ保存。アプリ再起動後も参照可能 |
| **データ容量** | CSV のローテーションは MVP では対象外。容量上限は設けない |
| **ポーリング間隔** | 1秒（タイマー表示の最大1秒ラグは許容） |
| **エラー時の挙動** | CSV 破損・uvicorn クラッシュ時の自動復旧は MVP では対象外。手動再起動で対応 |
| **同時接続** | 単一タブでの使用を前提。複数タブからの同時アクセスは非サポート |
| **配布** | フォルダをそのまま使う（exe 化不要） |
