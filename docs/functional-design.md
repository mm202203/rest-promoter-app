## データモデル定義

### TimerState（サーバーインメモリ）

タイマーの全状態を保持する。サーバー再起動時に初期化される（`accum_elapsed` も 0 にリセットされる点に注意）。

```python
class TimerState:
    is_running: bool        # タイマー動作中フラグ
    is_breaking: bool       # 休憩中フラグ（True のとき session_elapsed をカウントしない）
    is_first: bool          # first モードフラグ（起動時・リセット後に True）
    remaining: int          # カウントダウン残り秒
    timer_duration: int     # タイマー間隔（秒）。デフォルト 3600（60分）
    session_elapsed: int    # 現セッション経過秒（スヌーズ・休憩でリセット）
    accum_elapsed: int      # 累積作業秒（休憩時のみリセット）
    dialog_triggered: bool  # ダイアログ発火フラグ（UI がポーリングで検知）
    dialog_mode: str | None # 発火するダイアログのモード（first/timer/force）
    prev_task: str          # 前回の作業内容（次回ダイアログのデフォルト値）
    prev_load: int          # 前回の作業負荷（次回ダイアログのデフォルト値）
```

> **注意:** `dialog_mode = "self"` はサーバーが発火するものではなく、UI が自己申告ボタン押下時にクライアント側で直接ダイアログを開く。そのためサーバーの `dialog_mode` に `"self"` がセットされることはない。

**初期値:**

| フィールド | 初期値 |
|-----------|--------|
| `is_running` | `False` |
| `is_breaking` | `False` |
| `is_first` | `True` |
| `remaining` | `3600`（60分） |
| `timer_duration` | `3600` |
| `session_elapsed` | `0` |
| `accum_elapsed` | `0` |
| `dialog_triggered` | `False` |
| `dialog_mode` | `None` |
| `prev_task` | `""` |
| `prev_load` | `3` |

---

### ログ CSV スキーマ（`data/log.csv`）

| カラム名 | 型 | 内容 |
|----------|----|------|
| `timestamp` | datetime | 記録日時（ISO 8601形式） |
| `dialog_mode` | str | `first` / `timer` / `self` / `force` |
| `task` | str | 作業内容テキスト |
| `load` | int | 作業負荷（1〜5） |
| `state` | int | 状態スコア（1〜5） |
| `action` | str | `start`（firstモード） / `rest` / `skip` |
| `session_min` | int | セッション経過時間（分） |
| `accum_min` | int | 累積作業時間（分、記録時点） |
| `break_min` | int or null | 休憩時間（分）。`skip` / `start` 時は null。`force` モードは常に 15 |
| `snooze_min` | int or null | スヌーズ時間（分）。`rest` / `start` 時は null |

**`action` の値ルール:**

| `dialog_mode` | `action` | `break_min` | `snooze_min` |
|--------------|----------|------------|-------------|
| `first` | `"start"` | null | null |
| `timer` / `self` | `"rest"` | 5 / 10 / 15 のいずれか | null |
| `timer` / `self` | `"skip"` | null | 15 / 30 / 45 / 60 のいずれか |
| `force` | `"rest"` | 15（固定） | null |

**CSVサンプル:**

```csv
timestamp,dialog_mode,task,load,state,action,session_min,accum_min,break_min,snooze_min
2026-05-09T09:00:00,first,コーディング,4,4,start,0,0,,
2026-05-09T10:00:00,timer,コーディング,4,3,skip,60,60,,30
2026-05-09T10:30:00,self,コーディング,4,2,rest,30,90,10,
2026-05-09T10:45:00,timer,資料作成,3,4,skip,10,0,,60
2026-05-09T11:45:00,force,資料作成,3,2,rest,60,180,15,
```

**グラフ表示とデータの扱い:**

- **状態スコア推移グラフ**：`first` モードレコードを含む全レコードを使用（`first` モードでも状態スコアを記録するため）
- **連続作業時間グラフ**：`action = "start"` の `first` モードレコードは除外し、`session_min > 0` のレコードのみを使用

## API 設計

### エンドポイント一覧

| エンドポイント | メソッド | 概要 |
|----------------|----------|------|
| `GET /state` | GET | タイマー状態全体を返す |
| `POST /start` | POST | タイマー開始（`is_first = True` の場合は `first` ダイアログを発火） |
| `POST /pause` | POST | タイマー一時停止 |
| `POST /reset` | POST | タイマーリセット（`is_first` も初期化） |
| `POST /config` | POST | タイマー間隔の変更 |
| `POST /record` | POST | ダイアログの記録送信 |
| `POST /dialog/ack` | POST | `dialog_triggered` フラグのリセット |
| `GET /logs` | GET | ログ一覧取得 |

---

### `GET /state`

タイマーの現在状態をすべて返す。UI が 1秒ごとにポーリングする。

**レスポンス例:**
```json
{
  "remaining": 3542,
  "timer_duration": 3600,
  "session_elapsed": 58,
  "accum_elapsed": 4258,
  "is_running": true,
  "is_breaking": false,
  "is_first": false,
  "dialog_triggered": false,
  "dialog_mode": null,
  "prev_task": "論文読解",
  "prev_load": 5
}
```

---

### `POST /start`

タイマーを開始または再開する。

**リクエスト:** なし

**レスポンス:**
```json
{ "status": "started" }
```

**処理フロー:**
```
is_first が True の場合:
  → dialog_triggered = true, dialog_mode = "first" をセット
  → is_running は False のまま（タイマーはまだ開始しない）
  → タイマーの実際の開始は POST /record（first モード）完了後

is_first が False の場合:
  → is_running = true でタイマー再開（一時停止からの再開）
```

**異常系:**

| 条件 | レスポンス |
|------|-----------|
| `is_running = True` のとき（既に動作中） | `200 { "status": "already_running" }` |
| `is_breaking = True` のとき（休憩中） | `200 { "status": "breaking" }` |

---

### `POST /pause`

タイマーを一時停止する。

**リクエスト:** なし

**レスポンス:**
```json
{ "status": "paused" }
```

**異常系:**

| 条件 | レスポンス |
|------|-----------|
| `is_running = False` のとき | `200 { "status": "already_paused" }` |

---

### `POST /reset`

タイマーをリセットする。`is_first = True` に戻す。

**リクエスト:** なし

**レスポンス:**
```json
{ "status": "reset" }
```

**処理:**
- `remaining = timer_duration`
- `session_elapsed = 0`
- `is_first = True`
- `is_running = False`
- `is_breaking = False`
- `dialog_triggered = False`
- `dialog_mode = None`
- `accum_elapsed` はリセットしない（最後の休憩からの累積は継続）

---

### `POST /config`

タイマー間隔を変更する。進行中のタイマーには影響しない。

**リクエスト:**
```json
{ "duration_min": 45 }
```

**バリデーション:** `duration_min` は 1〜120 の整数。範囲外は 422 エラー。

**処理:**
- `timer_duration` を更新する
- `is_running = True` の場合でも `timer_duration` は更新するが `remaining` は変更しない
- 次回 `POST /reset` または `POST /record`（スヌーズ/休憩完了後のタイマー再起動）から新しい値が反映される

**レスポンス:**
```json
{ "status": "updated", "timer_duration": 2700 }
```

---

### `POST /record`

ダイアログの記録を送信する。ログに書き込み、状態を更新し、次タイマーをセットする。

**リクエスト:**
```json
{
  "dialog_mode": "timer",
  "task": "コーディング",
  "load": 4,
  "state": 3,
  "action": "rest",
  "break_min": 10,
  "snooze_min": null
}
```

**バリデーション:**

| フィールド | ルール |
|-----------|--------|
| `task` | 必須、空文字不可 |
| `load` | 1〜5 の整数 |
| `state` | 1〜5 の整数 |
| `action` | `"start"`（`dialog_mode = "first"` のとき）/ `"rest"` / `"skip"` |
| `break_min` | `action = "rest"` のとき必須。`timer`/`self` モードは 5/10/15 のいずれか。`force` モードは 15 のみ（それ以外は 422 エラー） |
| `snooze_min` | `action = "skip"` のとき必須（15/30/45/60 のいずれか） |

**処理フロー:**

```
dialog_mode = "first"（action = "start"）:
  → CSV にログ書き込み
  → prev_task, prev_load を更新
  → is_first = False
  → is_running = true でタイマー開始（ここで初めてカウントダウン開始）

action = "rest"（休憩）:
  → CSV にログ書き込み
  → accum_elapsed = 0、session_elapsed = 0
  → prev_task, prev_load を更新
  → is_breaking = true
  → remaining = break_min × 60 でタイマー開始（休憩タイマー）

action = "skip"（スヌーズ）:
  → CSV にログ書き込み
  → session_elapsed = 0（accum_elapsed はリセットしない）
  → prev_task, prev_load を更新
  → remaining = snooze_min × 60 でタイマー開始
```

**レスポンス:**
```json
{ "status": "recorded" }
```

---

### `POST /dialog/ack`

`dialog_triggered` フラグをリセットする。UI がダイアログを受け取ったことをサーバーに通知し、二重発火を防ぐ。

> **注意:** `self` モードは UI がクライアント側で直接ダイアログを開くため、このエンドポイントを呼ぶ必要はない。

**リクエスト:** なし

**レスポンス:**
```json
{ "status": "acknowledged" }
```

---

### `GET /logs`

ログ一覧を返す。グラフ・ログ表示用。

**クエリパラメータ:**
- `limit`（省略可）: 返す件数。省略時は全件

**ソート順:** `timestamp` 昇順（古い順）で返す。UI 側でグラフ描画（時系列順）・ログ表示（降順に逆転）の両方に対応する。

**レスポンス例:**
```json
{
  "logs": [
    {
      "timestamp": "2026-05-09T10:00:00",
      "dialog_mode": "timer",
      "task": "コーディング",
      "load": 4,
      "state": 3,
      "action": "skip",
      "session_min": 60,
      "accum_min": 60,
      "break_min": null,
      "snooze_min": 30
    }
  ]
}
```

## ダイアログ発火フロー

### サーバー起動ダイアログ（`first` モード）

```
[ユーザー操作]

POST /start を叩く（is_first = True の場合）
  → dialog_triggered = true, dialog_mode = "first" をセット
  → is_running は False のまま

[UI（app.js）]

GET /state の dialog_triggered = true を検知
  → POST /dialog/ack を叩く
  → first モードダイアログを表示

[ユーザーが first ダイアログ完了]

POST /record（action = "start"）を叩く
  → ログ書き込み・is_first = False
  → is_running = true でカウントダウン開始
```

### タイマー発火ダイアログ（`timer` / `force` モード）

```
[バックグラウンドスレッド（1秒ごと）]

is_breaking = false かつ is_running = true の場合:
  remaining -= 1
  session_elapsed += 1
  accum_elapsed += 1
  remaining が 0 になる
    → accum_elapsed が 10800秒（180分）以上: dialog_mode = "force"
    → それ以外: dialog_mode = "timer"
    → dialog_triggered = true をセット
    → is_running = false

[UI（app.js）- 1秒ポーリング]

GET /state の dialog_triggered = true を検知
  → POST /dialog/ack を叩く（二重発火防止）
  → dialog_mode に応じたダイアログを表示

[ユーザーが判断を送信]

POST /record を叩く
  → サーバー側でログ書き込み・状態更新・次タイマーセット
```

### 休憩終了後の自動ダイアログ（`timer` モード）

```
[バックグラウンドスレッド]

is_breaking = true の場合:
  remaining -= 1
  remaining が 0 になる
    → dialog_triggered = true, dialog_mode = "timer" をセット
    → is_breaking = false
    → is_running = false（UI がダイアログを表示するまで待機）

[UI（app.js）]

GET /state の dialog_triggered = true を検知
  → POST /dialog/ack を叩く
  → timer モードダイアログを表示（次の作業セッション開始フロー）
```

### 自己申告ダイアログ（`self` モード）

```
[ユーザー操作]

自己申告ボタンをクリック
  → UI が直接 self モードダイアログを表示（サーバーへのトリガー通知不要）
  → ダイアログ表示中はボタンを無効化

[ユーザーが判断を送信]

POST /record（dialog_mode = "self"）を叩く
  → 通常の record 処理と同様
```
