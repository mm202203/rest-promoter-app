# 技術仕様書

## テクノロジースタック

### バックエンド

| 項目 | 採用技術 | バージョン | 採用理由 |
|------|---------|-----------|---------|
| 言語 | Python | 3.11 以上 | 型ヒント・非同期処理が充実。uv との相性良好 |
| Webフレームワーク | FastAPI | 最新安定版 | 型ベースのバリデーション（Pydantic）、自動APIドキュメント生成、ASGI対応 |
| ASGIサーバー | uvicorn | 最新安定版 | FastAPI 推奨の ASGI サーバー。軽量で起動が速い |
| バリデーション | Pydantic | FastAPI に同梱 | FastAPI の依存として自動インストール。リクエストボディの型検証に使用 |
| データ処理 | pandas | 最新安定版 | CSV の読み書き・集計が簡潔に書ける |
| パッケージ管理 | uv | 最新安定版 | pip より高速。`pyproject.toml` ベースで依存関係を管理 |
| 標準ライブラリ | threading | Python 標準 | バックグラウンドスレッドと排他制御（Lock）に使用 |

### フロントエンド

| 項目 | 採用技術 | バージョン | 採用理由 |
|------|---------|-----------|---------|
| マークアップ | HTML5 | — | フレームワーク不要。シンプルな構成で十分 |
| スタイル | CSS3 | — | フレームワーク不要。カスタムプロパティ（CSS変数）で色管理 |
| スクリプト | Vanilla JavaScript | ES2020+ | 依存なし。ビルドステップ不要。ローカルツールとして適切 |
| グラフ描画 | Chart.js | 4.x（CDN・バージョン固定） | 折れ線・棒グラフに対応。設定が直感的で軽量 |

> **Chart.js のバージョン固定方針:** CDN 経由で読み込む際はマイナーバージョンまで固定する（例: `chart.js@4.4.0`）。バージョン未固定だと将来の自動更新で動作が変わるリスクがあるため。

### データストア

|------|---------|---------|
| ログ永続化 | CSV ファイル | pandas で直接分析可能。外部DBサーバー不要。人間が読める形式 |

## 起動スクリプト（`start.bat`）

```bat
@echo off
cd /d %~dp0
start "" http://localhost:8000
uv run uvicorn main:app --host 127.0.0.1 --port 8000
```

- `cd /d %~dp0`：`.bat` ファイルのあるディレクトリに移動（作業ディレクトリのずれを防ぐ）
- `start "" http://localhost:8000`：デフォルトブラウザでアプリを自動オープン
- `--host 127.0.0.1`：localhost のみバインド（外部アクセス不可）
- `--port 8000`：固定ポート

## アーキテクチャ方針

### レイヤー分離

```
routers/   → HTTPのルーティングのみ。ビジネスロジックを持たない
services/  → ビジネスロジックの中心。タイマー管理・判定・CSV操作
static/    → 表示とユーザー入力のみ。計算・判定をクライアント側に持たせない
```

この分離により、フェーズ2でのAI会話機能追加（`services/` への追加）が容易になる。

**例外：`self` モードのトリガー**
自己申告ダイアログのトリガーのみ、クライアント側（`app.js`）が直接ダイアログを開く。
これはユーザーが明示的にボタンを押した結果であり、サーバー側でモードを判定する必要がないため。
ダイアログへの入力送信（`POST /record`）は通常どおりサーバー側で処理するため、判定ロジックがクライアントに漏れるわけではない。

### 状態管理方針

- **タイマー状態はサーバーのインメモリ（シングルトン）で管理**
  - ページリロード・タブ切り替えに対して状態が保持される
  - サーバー再起動時はリセットされる（MVP 仕様）
- **ログのみ CSV に永続化**
  - アプリ再起動後もログの閲覧・グラフ表示が可能
- **UIはレンダリングと入力受付のみ**
  - タイマー計算・モード判定・アドバイス分岐はすべてサーバー側で処理
  - 例外は上記の `self` モードトリガーのみ

### スレッド設計

```
[メインスレッド]
  FastAPI + uvicorn がリクエストを処理

[バックグラウンドスレッド（1本）]
  アプリ起動時（main.py の lifespan イベント）に threading.Thread で起動し常駐
  is_running / is_breaking フラグで動作を制御（False のときはスリープしてループ）
  1秒ごとに TimerState を更新し、タイマー発火判定・フラグセットを行う

[スレッド間の排他制御]
  threading.Lock で TimerState への書き込みを保護
  Lock の保持スコープは「TimerState のフィールドを変更する操作のみ」に限定する
  ループ全体をロックしない（読み取り専用の GET /state を不必要にブロックしない）
```

### 静的ファイル配信と CORS

- FastAPI の `StaticFiles` を `main.py` でマウントし、`static/` ディレクトリを配信する
- ブラウザは `http://localhost:8000` から `index.html` を受け取り、同じオリジンの `/state` `/record` 等を呼ぶ
- 同一オリジンのリクエストのみのため **CORS 設定は不要**

## 技術的制約と要件

### 制約

| 制約 | 内容 |
|------|------|
| ネットワーク | localhost のみ。外部サービスへの通信なし（MVP） |
| ストレージ | CSV ファイルのみ。外部データベース不使用 |
| 認証・認可 | なし（個人利用・ローカルのみ） |
| ビルドステップ | なし。バンドラー・トランスパイラ不使用 |
| 同時接続 | 単一タブ前提。複数タブは非サポート |

### 依存関係の方針

- **フロントエンドの外部ライブラリは CDN 経由のみ**（Chart.js）
  - ネットワーク接続がない環境では動作しないが、MVP では許容
  - 将来的に必要であればローカルにダウンロードして `static/` に配置
- **バックエンドの依存は最小限**（fastapi / uvicorn / pandas の3つのみ）

## パフォーマンス要件

| 項目 | 要件 | 根拠 |
|------|------|------|
| ポーリング間隔 | 1秒 | タイマー表示の最大1秒ラグを許容 |
| `GET /state` レスポンスタイム | 100ms 以内 | Lock 取得後のインメモリ読み取りのみ。Lock 保持スコープを書き込みに限定することでブロック時間を最小化する |
| `POST /record` レスポンスタイム | 500ms 以内 | CSV 追記書き込みを含むが少量データのため十分達成可能 |
| `GET /logs` レスポンスタイム | 1秒 以内 | pandas での CSV 読み込みを含む。1日最大30件×365日≒10,000行/年の規模では pandas は十分高速（通常数十ms） |
| サーバー起動時間 | 5秒 以内 | uvicorn の起動は通常 1〜2秒。ブラウザ自動オープンと並行して起動 |
| メモリ使用量 | 特に規定なし | TimerState はごく小さいオブジェクト。CSV も数年分でも数MB程度 |

> **計測方針:** 個人ツールのため厳密な計測は不要。体感でタイマー表示のラグや操作の遅延が気になる場合にブラウザの DevTools（Network タブ）で確認する。

## セキュリティ要件

個人利用・ローカル環境のため、外部公開を前提としたセキュリティ対策は不要。
ただし最低限以下を遵守する。

| 項目 | 対応 |
|------|------|
| バインドアドレス | `127.0.0.1` のみ（LAN内の他端末からアクセス不可） |
| 入力バリデーション | FastAPI + Pydantic モデルによるサーバー側バリデーション（型・範囲・必須チェック） |
| XSS 対策 | ユーザー入力を DOM に挿入する際は `textContent` プロパティを使用する。`innerHTML` は使用しない。Chart.js のグラフラベルに `task` を使う場合も Chart.js の `label` プロパティ経由（内部でエスケープ済み）に限定する |
| CSV インジェクション | `task` フィールドの先頭文字が `=` `+` `-` `@` の場合、CSV 書き込み時にシングルクォート（`'`）を先頭に付加する。Excel で開いた際に数式として実行されることを防ぐ |

## フェーズ2への拡張ポイント

現設計は以下の拡張を想定して分離されている。

| フェーズ2機能 | 対応する拡張ポイント |
|-------------|-------------------|
| AI会話型インターフェース | `services/session_service.py` にClaude API呼び出しを追加 |
| WebSocket対応 | `routers/timer.py` にWebSocketエンドポイントを追加。ポーリングと共存可能 |
| CSV出力UI | `routers/log.py` にダウンロードエンドポイントを追加 |
| 音声入力 | `static/app.js` にWeb Speech APIを追加 |
| 散歩リマインダー | バックグラウンドスレッドに散歩タイマーロジックを追加 |
